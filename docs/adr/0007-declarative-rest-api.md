# Resource-oriented REST API, declarative writes only, async by construction

The Control Plane Core's single public contract is resource-oriented REST/JSON: `PUT /v1/{kind}/{name}` applies a full Spec, `GET` returns Spec+Status, `DELETE` marks for deletion. Two disciplines are load-bearing: (1) **declarative writes only** — no imperative verbs (`/restart`, `/redeploy`); clients state what should exist, never what to do; (2) **async by construction** — writes return `202 Accepted` once Spec persists, and convergence is observed later via Status. This forces every client (Portal included) to be honest about eventual consistency. The CLI (`cpctl`) is a plain API client with no privileged path into the core. gRPC was rejected (Go/TS codegen friction, nothing curl-able in a live demo); CLI-only was rejected (the Portal must be a separate process per ADR-0003).
