# aws-s3-self-service is a quarry and a deliberate parallel, not an integration target

The sibling project `~/dev/profile-projects/aws-s3-self-service` (golden-path S3 provisioning, OIDC, Step Functions → CodeBuild Terraform runner) stays a separate project. We **quarry** its design artifacts — its OIDC/auth ADRs inform this project's multi-user milestone, and its hardened-S3-bucket golden-path spec is the blueprint for the future AWS Provider's first resource kind — and we cultivate the **deliberate parallel**: two architectures for the same self-service problem class (pipeline-driven vs reconciliation-driven), a compare-and-contrast that is itself interview material. Early integration was rejected: that project is Terraform-runner-shaped, not reconciler-shaped, and welding two greenfield designs couples them prematurely. Later integration stays possible precisely because the Provider contract (ADR-0001) doesn't care what sits behind it.
